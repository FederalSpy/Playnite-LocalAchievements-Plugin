# Contributing / Contribuciones

## English
Thanks for helping.

### Workflow
1. Fork the repo.
2. Create a branch: `feature/my-change`.
3. Make small, focused commits.
4. Open a Pull Request with clear context.

### Guidelines
- Keep changes scoped and easy to review.
- Prefer readability over clever tricks.
- If touching UI, include before/after screenshots when possible.
- If changing behavior, explain user impact.

### Style
This project is a learning project and the code quality is uneven.
Feel free to improve structure incrementally, but avoid huge rewrites in a single PR unless discussed first.

## Español
Gracias por ayudar.

### Flujo
1. Haz fork del repo.
2. Crea una rama: `feature/mi-cambio`.
3. Haz commits pequeños y enfocados.
4. Abre un Pull Request con contexto claro.

### Guías
- Mantén cambios acotados y fáciles de revisar.
- Prioriza legibilidad sobre trucos complejos.
- Si tocas UI, incluye capturas antes/después cuando sea posible.
- Si cambias comportamiento, explica el impacto al usuario.

### Estilo
Este proyecto es de aprendizaje y la calidad del código es irregular.
Puedes mejorar la estructura poco a poco, pero evita reescrituras masivas en un solo PR sin discutirlo primero.
